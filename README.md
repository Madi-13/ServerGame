# ServerGame (C)

Учебный проект **клиент–серверной игры** на языке C: отдельные программы **сервер** и **клиент**, плюс файл карты/уровня `map.txt`.

В репозитории лежат:
- `server.c` — серверная часть
- `client.c` — клиентская часть
- `map.txt` — данные карты/поля
- `README.md` :contentReference[oaicite:0]{index=0}

---

## Что это такое

**ServerGame** — минимальный пример сетевого взаимодействия в стиле “игра по сети”:
- сервер запускается отдельно и ждёт подключения клиентов;
- клиент подключается к серверу и участвует в игровом процессе;
- карта/сценарий берётся из `map.txt`.

> Точные правила игры/формат сообщений зависят от реализации в `server.c` и `client.c`.

## Быстрый старт

### 1) Склонировать репозиторий
```bash
git clone https://github.com/Madi-13/ServerGame.git
cd ServerGame
````

### 2) Собрать сервер и клиент

```bash
gcc -Wall -Wextra -O2 -o server server.c
gcc -Wall -Wextra -O2 -o client client.c
```

> Если у тебя macOS и ругается на какие-то заголовки/флаги — просто убери лишние предупреждения и собери ещё раз.

### 3) Запустить сервер

```bash
./server
```

### 4) Запустить клиента(ов) в другом терминале

```bash
./client
```

Если клиенту нужен IP/порт — проверь значения прямо в исходниках (обычно это константы в начале файла) и при необходимости поменяй.

## Как работает (логика на уровне проекта)

### Компоненты

* **Сервер (`server.c`)**

  * принимает подключения;
  * хранит состояние игры;
  * обрабатывает действия игроков;
  * рассылает обновления клиентам.
* **Клиент (`client.c`)**

  * подключается к серверу;
  * отправляет команды/ввод игрока;
  * получает и отображает состояние игры.
* **Карта (`map.txt`)**

  * задаёт поле/уровень (формат зависит от реализации).
